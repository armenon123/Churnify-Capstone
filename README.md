# 📡 Telecom Customer Churn Predictor
### Grade 12 AI Final Project

Predict whether a telecom customer is likely to cancel their subscription, using Exploratory Data Analysis and three machine learning models.

---

## Project Structure

```
hs-churn-predictor/
├── 1_EDA.ipynb                    ← Notebook 1: Data exploration & preprocessing
├── 2_Model_Building.ipynb         ← Notebook 2: Model training, evaluation & comparison
├── app.py                          ← Streamlit web app
├── train_models.py                 ← Run once to generate model files for the app
├── requirements.txt
├── README.md
└── models/                         ← Auto-generated by train_models.py
```

> `tel_churn.csv` is produced by `1_EDA.ipynb` and consumed by `2_Model_Building.ipynb` — run them in order.

The dataset (`WA_Fn-UseC_-Telco-Customer-Churn.csv`) lives one folder up and is referenced as `../WA_Fn-UseC_-Telco-Customer-Churn.csv`.


---

## Running the Project

### 1. Run the notebooks in order

```bash
jupyter notebook
```

Open **`1_EDA.ipynb`** first and run all cells (`Kernel → Restart & Run All`).  
Then open **`2_Model_Building.ipynb`** and do the same.

> `1_EDA.ipynb` saves `tel_churn.csv` which `2_Model_Building.ipynb` needs.

### 2. Train and save the models (for the app)

```bash
python train_models.py
```

This trains Logistic Regression, KNN, and Decision Tree on the dataset and saves them to `models/`. Only needs to be run once.

### 3. Launch the Streamlit app

```bash
streamlit run app.py
```

Opens in your browser at `http://localhost:8501`

## App Features

| Tab | What it does |
|-----|-------------|
| **Predict Single Customer** | Fill in a form with customer details → get a churn prediction with confidence % |
| **Batch Prediction** | Upload a CSV file → download predictions for all customers |
| **Model Comparison** | Bar chart comparing all three models' accuracy and F1 score |

---

## Models Used

| Model | Accuracy | F1 Score |
|-------|----------|----------|
| Logistic Regression | ~73% | ~61% |
| KNN (K=5) | ~76% | ~56% |
| Decision Tree (depth=5) | ~73% | ~62% |

> **Note on imbalance:** The dataset has ~73% non-churners and ~27% churners.  
> We use `class_weight='balanced'` so models don't just predict "No Churn" for everyone.

---

## Dataset

**IBM Telco Customer Churn** — 7,043 customers, 21 columns  
Source: [Kaggle](https://www.kaggle.com/datasets/blastchar/telco-customer-churn)

Key columns: `tenure`, `MonthlyCharges`, `TotalCharges`, `Contract`, `InternetService`, `PaymentMethod`, `Churn` (target)
